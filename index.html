Swedish Railways (SJ) - My train design
________

#: UX Design, Wireframing, Usability testing, Sketch, Prototyping, Android

My train is an application used by the Train inspectors to check travellers tickets, look up current train situations, current bookings and a lot more.

This project was an initiative to re-design their current application that was a Hybrid web-application into a Native solution. The goal was to take advantage of native system speed, native components and familiar UX and UI to create an Application that listened to the workers needs and wants into a neat package of a new Android App.

User feedback on the app: "The app just beceoms better and better. Super convenient ticket inspection from the last updates. Goes alot smoother and quicker now. Very well done! ❤"

Mad Sand - AR 2 player Game
________

#: Unity, Kinect, Augmented Reality, C# programming, Interaction Design

Mad Sand is a game taking augmented reality gaming to a new level, allowing you to shape a digital world in real time through a physical sandbox. The game is an asymmetric cooperative game, where one player (“the Sandmaker") shapes the sand landscape to help the other player (“the Penguin”) to collect sweets. Sweets can appear floating high up in the sky or deep beneath the earth, thus the Sandmaker must change the landscape so that the Penguin successfully can grab the sweets. To help the Sandmaker see what's going on an augmented reality experience is also provided through a projection of the game world onto the sand.

This project was a part of the Advanced graphics and interaction 2015 course at KTH.

Snickers Workwear - Kneepad sensor
________

#: UX Design, Wireframing, Prototyping, Sketch, iOS, Bluetooth sensors, Wearable technology

Snickers Workwear excel in making world class knee pads for workers within their business. The project consisted of creating a mobile application with the help of a kneepad sensor that was to be attached close to the knee. With the help of the sensor the application could gather information on, how many times the user has bent the knee, amount of time on knees as well as temperature and sound levels.

The goal was experimenting with new technology, being in the forefront to don't lag behind and to add value to their current line of products.

Other Room - Browser WebGL graphics
________

#: WebGL, Three.js, HTML, CSS

Other Room purpose was to build a meditative scene that users could project on a wall at home to create the illusion of a another room just beyond the wall. To avoid creating a stressing experience the interaction would consist only of sound, the scene slightly changing depending on the current volume. Thus the user could put on their favorite harmonic music and watch the assets of the other room sway to the gentle music.

This project was a part of the Advanced graphics and interaction 2015 course at KTH.

Building Blocks - Physical interaction
________

#: Arduino, Sensors, pySerial, Interaction Design

We wanted to create toys with physical interaction for children 1-4 years old. Building blocks have long been a robust toy for children with little evolvement for so many years, so we decided to make an alternate version of them. To inspire creativity and curiousness the blocks will change colors depending on the height of the tower the children are building. To achieve each block has a multicolor LED and Bluetooth buit in, together with a pressure sensor and pySerial server this magic was possible.

This project was part of the Physical Interaction course 2015 at KTH.
